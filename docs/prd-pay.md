# PRD — 결제 화면 (진단 리포트 결제)

## 1. 목적
- 진단 리포트 결제 단계에서 결제 수단 선택, 약관 동의, 결제 실행을 단일 화면에서 완료한다.

## 2. 주요 사용자 시나리오
- 상단에서 매물 정보를 확인한 뒤, 결제 요약(금액/할인/총 결제금액)을 보고, 결제 수단(직방 페이)과 카드, 할부 조건을 선택한 뒤 필수 약관에 동의하고 결제한다.

## 3. 필수 기능
- 선택한 매물 정보 (화면 상단에 표시)
- 결제 요약 표시: 리포트 금액, 할인, 총 결제금액(예: 29,900원)
- 결제 수단 선택: 기본 ‘직방 페이’ 노출(라디오)
- 카드 선택: 카드 리스트 중 하나 선택(예: 신한카드, 우리카드)
- 할부 선택: 드롭다운(일시불 기본)
- 약관 동의: 필수 2개, 선택 1개
- CTA: 총 결제금액 표시 + 결제하기

## 4. UI Components
- **Item info 매물 정보 (최상단)**
  - 항목: img thumbnail, 상품카테고리(아파트/원룸/빌라/오피스텔), 단지명(아파트인 경우), 주소, 보증금/월세(있는경우)
- **Summary 카드**
  - 항목: 리포트 금액, 할인, 총 결제금액
  - 통화 표기: 원
- **PaymentMethod (라디오)**
  - 기본값: 직방 페이
- **CardSelector (단일 선택 리스트)**
  - 카드 항목: 카드명/마스킹 PAN 표기
  - 선택 시 강조 스타일
- **InstallmentSelector (드롭다운)**
  - 옵션: 일시불 (필수)
- **Checkbox (약관 동의)**
  - 필수1: “[필수] 서비스 이용약관 및 개인정보처리방침에 동의합니다.”
  - 필수2: “[필수] 결제 대행 서비스 약관에 동의합니다.”
  - 선택: “[선택] 마케팅 정보 수신에 동의합니다.”
- **CTA**
  - 라벨: “29,900원 결제하기” (총 결제금액 반영)
  - disabled 조건: 필수 약관 미동의 시 비활성

## 5. 입력/검증 로직 (최소)
- 결제 수단: 필수 (기본 직방 페이)
- 카드: 하나 필수 선택
- 할부: 필수 선택(일시불 기본)
- 약관: 필수 2개 모두 체크, 선택 1개는 자유
- 위 조건 충족 시 CTA 활성, 아니면 비활성

## 6. 상태/피드백
- 비활성: 필수 약관 미동의 시 CTA 비활성
- 결제 실행: 결제 호출/모달/다음 단계는 별도 플로우에서 처리

## 7. 전환/동작
- CTA 클릭 시 결제 프로세스/PG 호출(또는 직방 페이 플로우)로 진입
- 뒤로 가기: 상단 네비 백으로 이전 화면 복귀

## 8. 제외 범위
- PG 연동 상세, 실제 결제/취소/실패 처리, 카드 추가/삭제, 프로모션/쿠폰, 영수증/매출전표 UI

## 9. 측정 지표 (예시)
- 결제 CTA 클릭률
- 결제 완료율
- 약관 미동의로 인한 이탈률
