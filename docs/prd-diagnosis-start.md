# PRD — 진단 시작 화면

## 1. 목적
- 사용자가 매물 진단을 요청하기 전, 필수 정보(주소, 면적, 보증금, 계약 기간)와 약관 동의를 최소 단계로 수집한다.
- 입력 완료 시 “진단 시작하기” CTA를 통해 결제/진단 요청 단계로 진입한다.

## 2. 주요 사용자 시나리오
- 사용자는 주소·면적·보증금·계약 기간을 입력(또는 선택)하고 필수 약관에 동의한 뒤 “진단 시작하기”를 누른다.
- 필수 항목 미입력 또는 필수 약관 미동의 시 CTA는 비활성 상태를 유지한다.

## 3. 필수 기능
- 주소 입력: 텍스트 입력 (필수)
- 면적 입력: 숫자 입력, m² 단위 표기 (필수)
- 보증금 입력: 숫자 입력, “만원” 단위 표기 (필수)
- 월세 입력: 숫자 입력, “만원” 단위 표기 (선택)
- 계약 기간 선택: 세그먼트(1년/2년/3년 이상) 중 택1 (필수)
- 약관 동의: 필수 2개, 선택 1개 체크박스
- CTA: “진단 시작하기” 버튼 (필수 입력/동의 충족 시 활성)
- 요약/가격 표시: “진단 리포트 29,000” 및 “리포트 생성까지 최대 20분 소요” 안내

## 4. UI Components (디자인 시스템 / 코드 매핑)
- **Input (공통)**
  - 주소/면적/보증금/월세 모두 단일 Input 컴포넌트 사용
  - 구분: `keyboardType=default`(텍스트), `keyboardType=numeric`(숫자)
  - suffix 필요 시: `suffixEnabled=true`, `suffixText="m²"` 또는 `"만원"`
  - placeholder 예시: 주소 “서울시 강남구 역삼동 123-45”, 면적 “84”, 보증금 “50000”, 월세 “50”
  - required/disabled 여부는 컴포넌트 prop이 아니라 **화면 로직**에서 결정
- **SegmentedControl (계약 기간)**
  - 옵션 3개 고정: “1년”, “2년”, “3년 이상”
  - 선택 상태(value) 관리: 화면 로직에서 주입/변경
  - required/disabled 여부는 컴포넌트 prop이 아니라 **화면 로직**에서 결정
- **Checkbox (약관 동의)**
  - 필수1: “[필수] 개인정보 수집 및 이용 동의”
  - 필수2: “[필수] 서비스 이용약관 동의”
  - 선택: “[선택] 마케팅 정보 수신 동의”
  - checked/disabled 여부는 **화면 로직**에서 결정
- **Button.Primary (CTA)**
  - label: “진단 시작하기”
  - disabled: 주소/면적/보증금/계약 기간 미충족 또는 필수 약관 미동의일 때 화면 로직에서 계산 후 prop 전달
- **Summary / Price**
  - 텍스트: “진단 리포트 29,000”
  - 서브 텍스트: “리포트 생성까지 최대 20분 소요”

## 5. 입력/검증 로직 (최소)
- 주소: 빈 값 불가
- 면적: 빈 값 불가, 숫자만 허용 (numeric 키패드)
- 보증금: 빈 값 불가, 숫자만 허용 (numeric 키패드), 만원 단위
- 월세: 선택 입력, 숫자만 허용 (numeric 키패드), 만원 단위
- 계약 기간: 1년/2년/3년 이상 중 하나 필수 선택
- 약관: 필수 2개 모두 체크 필수, 선택 1개는 자유
- 위 조건 모두 충족 시 CTA 활성, 아니면 비활성

## 6. 상태/피드백
- 필수 미충족 시 CTA 비활성 (추가 에러 메시지 노출은 불필요)
- 별도 로딩/에러 상태 없음 (다음 단계 이동 시 처리)

## 7. 전환/동작
- CTA 클릭 시 다음 단계(결제/진단 요청 플로우)로 이동

## 8. 제외 범위
- 주소 자동완성, 주소 검증, 상세 Validation, 에러 메시지, 다중 Variant 버튼, 추가 입력 필드

## 9. 측정 지표 (예시)
- 입력 완료 → CTA 활성 비율
- CTA 클릭률(다음 단계 전환률)
