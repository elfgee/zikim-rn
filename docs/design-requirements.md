# DESIGN: 지킴진단 - 디자인 요건(UX/UI)

본 문서는 `PRD_지킴진단.md` 및 연결된 지킴진단 PRD 문서를 기반으로, 구현/데이터/API를 제외한 **디자인(UX/UI) 요건만** 재정리한 문서입니다.

---

## 1. 문서 범위

## 1.1 포함 범위

- 사용자 여정에 포함되는 화면의 UI 구성 요소
- 화면별 상태(정상/빈 상태/오류/지연)에서의 문구 및 표시 방식
- 리포트 페이지 공통 요약 및 탭별 정보 구조(섹션, 카드, 차트, 지도 등)

## 1.2 제외 범위

- API 설계, DB/데이터 타입, 연동 방식(웹소켓/폴링 등) 등 구현 상세
- 백오피스 권한/승인 프로세스 등 운영 정책의 상세 설계

---

## 2. 유지해야 할 User Flow

## 2.1 사용자(앱) 플로우

```
[직방 APP 게이트웨이]
        ↓
[지킴진단 버튼 클릭]
        ↓
[거래 정보 입력 화면]
        ↓
[주소 검색 & 리포트 히스토리 화면]
        ├─→ [주소 검색] → [결제 화면] → [발급 진행중] → [리포트 페이지]
        └─→ [리포트 히스토리] → [기존 리포트 조회]

[리포트 페이지 구성]
        ├─ 리포트 요약 (공통)
        ├─ 매물 진단 (탭)
        ├─ 집주인 진단 (탭)
        ├─ 시세 진단 (탭)
        ├─ 대출/보험 예비심사 (탭)
        ├─ 특약 (탭)
        ├─ 범죄/치안 (탭)
        └─ 생활/편의 (탭)
```

---

## 3. 공통 UI 규칙

## 3.1 상태 태그/배지

- 양호
- 확인 필요
- 위험
- 가능성 높음
- 불가
- 해당 없음
- 확인 불가

## 3.2 상태 색상(명시된 범위만)

- 양호: 녹색 라벨/표시
- 확인 필요: 문서별로 빨간색 표시가 명시된 섹션 존재(예: 시세 진단)
- 위험: 색상 규칙은 문서에 구체 명시 없음(확인 필요)
- 가능성 높음: 녹색 배지
- 불가: 빨간색 배지

## 3.3 금액 표기

- 리포트 요약/시세 진단 등에서 금액은 천만원 단위로 요약 노출
- 거래 정보 입력 화면에서 금액 입력 시 숫자 입력값과 한글 변환 결과를 함께 표시
- 예시: 200000000 입력 → "200,000,000원 (2억원)"

## 3.4 접기/펼치기(디스클로저)

- 리스트형 항목의 상세 정보는 하위 정보로 제공
- 하위 정보 영역은 접을 수 있어야 함

---

## 4. 화면별 디자인 요건 (사용자 플로우)

## 4.1 게이트웨이 화면

## 4.1.1 진입점(버튼) 구성

- 버튼 형태로 "지킴진단" 제목 노출
- 서비스 매력을 나타내는 서브 문구 노출
- 서브 문구 예시: "거래 전 알아야 할 권리분석, 치안·안전, 생활편의 한눈에"

## 4.1.2 내비게이션

- 지킴진단 버튼 클릭 시 거래 정보 입력 화면으로 이동

---

## 4.2 거래 정보 입력 화면

## 4.2.1 입력 UI 구성

- 거래 목적 선택: 전세, 월세, 매매 (필수)
- 전세 선택 시: 전세금 입력 (필수)
- 월세 선택 시: 보증금 입력 (필수)
- 월세 선택 시: 월세 입력 (필수)
- 매매 선택 시: 매매금액 입력 (필수)
- 전세/월세 선택 시: 계약 기간 입력(필수)
- 매매 선택 시: 계약 기간 입력 필드 비노출

## 4.2.2 계약 기간 입력 방식

- 선택형 옵션 제공(예: 1년, 2년 등)
- 직접 입력 옵션 제공

## 4.2.3 진행 제어(CTA)

- 필수 입력 항목 미충족 시 다음 단계 진행 불가
- 모든 필수 입력 항목 충족 시 다음 단계(주소 검색 화면) 진행 가능

## 4.2.4 거래 목적 변경 UX

- 거래 목적 변경 시 기존 입력값 초기화
- 거래 목적에 따라 입력 필드 구성이 즉시 전환

## 4.2.5 입력 검증 메시지(예시 문구)

- 거래 목적 미선택: "거래 목적을 선택해주세요"
- 금액 미입력: "금액을 입력해주세요"
- 전세/월세 기간 미선택: "계약 기간을 선택해주세요"
- 숫자 외 입력: 숫자만 입력 가능하도록 제한(문자 입력 방지)

---

## 4.3 주소 검색 & 리포트 히스토리 화면

## 4.3.1 주소 검색 UX

- 도로명 기준 검색
- 초성 검색 지원
- 입력 중 자동완성 목록 제공
- 검색 결과는 입력값과 일치하는 도로명 목록으로 표시
- 검색 결과 선택 시 선택한 주소가 입력 박스에 표시

## 4.3.2 동/호 선택 UX

- 선택한 주소에 동/호 정보가 있는 경우 같은 화면에서 동/호 선택 UI 표시
- 동/호 선택 완료 후 동/호가 포함된 주소가 입력 박스에 표시

## 4.3.3 검색 결과 없음/오류

- 검색 결과가 없는 경우 안내 메시지 표시 및 검색어 수정 유도
- 네트워크 오류 시 오류 메시지 표시 및 재시도 옵션 제공

## 4.3.4 리포트 히스토리 진입

- 주소 검색 화면에서 "기존 발급 내역" 버튼을 통해 히스토리 진입

---

## 4.4 리포트 히스토리(기존 발급 내역)

## 4.4.1 목록 기본 동작

- 화면 진입 시 자동으로 기존 발급 내역 조회 후 목록 표시
- 정렬: 최신순

## 4.4.2 목록 아이템 표시 정보

- 리포트 발급일시
- 매물 주소
- 거래 목적(전세/월세/매매)
- 리포트 상태(발급 완료/진행중/실패)

## 4.4.3 빈 상태

- 발급 내역이 없는 경우 빈 상태 안내 메시지 표시
- "새 리포트 발급하기" 안내 포함

## 4.4.4 액션

- 목록에서 리포트 선택 시 해당 리포트 페이지로 이동
- 리포트 공유 기능 제공

---

## 4.5 결제 화면

## 4.5.1 결제 옵션 UI

- 1회: 5,500원
- 5회: 22,000원
- (잔여) 이용권: 사용 가능한 잔여 횟수 표시

## 4.5.2 결제 흐름

- 1회/5회 선택 시: 결제 모듈 표시 후 결제 완료 시 발급 진행중 화면으로 이동
- (잔여) 이용권 선택 시: 결제 과정 없이 바로 발급 진행중 화면으로 이동

## 4.5.3 이용권 안내 문구

- 이용권은 구매 후 사용 횟수를 제외한 잔여 횟수가 남아있는 상태임
- 이용권은 직방 타 서비스 이용 또는 마케팅 목적으로 무료 제공될 수 있음

---

## 4.6 발급 진행중 화면

## 4.6.1 공통 구성

- 발급 진행 상태 표시(진행중 애니메이션 또는 프로그레스 바)
- 예상 소요 시간 안내(예: "약 30초 정도 소요됩니다")
- 화면 이탈 안내(예: "화면을 벗어나도 발급이 완료되면 알림으로 알려드릴게요")

## 4.6.2 상태별 화면

## 4.6.2.1 진행중

- 안내 문구: "리포트를 발급하고 있어요"
- 예상 소요 시간: "약 30초 정도 소요됩니다"

## 4.6.2.2 완료

- 안내 문구: "리포트 발급이 완료되었습니다!"
- 리포트 페이지로 자동 이동 또는 "리포트 보기" 버튼 제공

## 4.6.2.3 실패

- 안내 문구: "리포트 발급에 실패했습니다"
- 이용권 차감 취소 안내: "이용권은 차감되지 않았습니다"
- 재시도 버튼 제공
- 고객센터 문의 버튼 제공

## 4.6.3 장시간 대기(예상 시간 초과)

- 안내 문구: "발급이 평소보다 오래 걸리고 있어요. 잠시만 더 기다려주세요."

## 4.6.4 네트워크 오류

- 재연결 안내 메시지 표시
- 네트워크 복구 후 발급 상태 자동 확인

---

## 5. 리포트 페이지 디자인 요건

## 5.1 리포트 요약(공통)

## 5.1.1 매물 헤더(기본 정보)

- 주소 노출(사용자가 선택한 주소)
- 거래 유형 노출(매매/전세/월세)
- 거래 유형에 맞는 금액 정보 노출
- 금액 표기는 천만원 단위로 노출
- 전세/월세의 경우 계약 기간 노출
- 매매의 경우 계약 기간 미노출
- 기간 정보가 없는 경우 요약 영역 미표시

## 5.1.2 AI 종합진단

- 종합 요약 성격을 드러내는 라벨(배지) 노출
- 예시 라벨: "AI 종합 진단 의견"
- 종합 요약 문장은 리포트의 핵심 결론을 자연어 문장으로 제공
- 종합 요약 문장은 여러 사실을 포함할 수 있음
- 핵심 구절이 강조되어 보일 수 있어야 함

## 5.1.3 점검 항목 요약 카드(탭 요약)

- 매물 진단 요약 상태 노출
- 집주인 진단 요약 상태 노출
- 시세 진단 요약 상태 노출
- 대출/보험 진단 요약 상태 노출
- 범죄/치안 요약 상태 노출
- 생활/편의 요약 상태 노출

## 5.1.4 요약 카드 표기 규칙(이상 항목/양호)

- 이상이 있는 경우 이상 항목 개수 표시
- 이상이 없는 경우:
- 매물/집주인/시세/대출보험은 0개로 표시
- 범죄/치안, 생활/편의는 "양호"로 표시

## 5.1.5 특약 요약 카드

- 안내 문구: "이 집 맞춤형 특약도 확인해보세요!" 성격의 문구 제공
- 추천 특약 개수 노출

## 5.1.6 고지/주의 문구

- 요약 영역에 고지/주의 문구 노출(문구 내용은 PRD에 구체 미기재)

---

## 5.2 탭: 매물진단

## 5.2.1 탭 구성

- AI 요약 영역(라벨: "집 위험도 분석 AI 요약")
- 건축물대장 분석 영역(위반건축물)
- 권리관계 분석 영역(12개 항목 리스트)

## 5.2.2 AI 요약 UI

- AI 요약 성격을 명확히 드러내는 라벨 노출
- 등기부 위험 권리 및 보증금 회수 위험도에 대한 종합 요약 문장 제공

## 5.2.3 건축물대장 분석 UI

- 위반건축물 여부 상태 노출
- 상태 표기: "양호" 또는 "해당 없음" 등

## 5.2.4 권리관계 분석 리스트 공통 규칙

- 항목들을 리스트 형태로 노출
- 각 항목별 상태 표시
- 상태가 양호한 경우 녹색 라벨로 표시
- 항목 상세 정보는 하위 정보로 제공
- 하위 정보 영역은 접을 수 있어야 함

## 5.2.5 권리관계 분석 항목(12개)

- 대지권
- 토지별도등기
- 가등기
- 압류/가압류
- 처분금지가처분
- 신탁
- 경매개시결정
- 근저당권
- 임차권등기명령
- 전세권
- 민간임대주택등록 여부
- 등기사항 변경예정

## 5.2.6 항목별 출력 패턴(공통)

- 상태 태그(예: "양호", "확인 필요", "위험")
- 상태 태그와 함께 "있음/없음/등록됨" 등 상세 표기
- 하위 정보: 정의, 설명, 특약 안내

## 5.2.7 데이터 부재/부분 실패 표시

- 등기부등본 정보 없음: 해당 정보 없음 안내 표시
- 건축물대장 정보 없음: 해당 정보 없음 안내 표시
- 일부 항목 정보 미조회: 해당 항목은 "확인 불가"로 표시

---

## 5.3 탭: 집주인 진단

## 5.3.1 탭 구성

- AI 요약 영역(라벨: "집주인 리스크 AI 요약")
- 소유자 정보 영역(소유구분, 소유자)
- 소유주 상세 분석 영역(4개 항목 리스트)

## 5.3.2 소유주 상세 분석 리스트 공통 규칙

- 항목들을 리스트 형태로 노출
- 각 항목별 상태 표시
- 상태가 양호한 경우 녹색 라벨로 표시
- 항목 상세 정보는 하위 정보로 제공
- 하위 정보 영역은 접을 수 있어야 함

## 5.3.3 소유주 상세 분석 항목(4개)

- 건물, 토지 소유자 일치
- 임대사업자 등록
- 보증금미반환 이력
- 고액 상습 체납자 조회

## 5.3.4 고액 상습 체납자 조회 UI

- 조회 버튼 제공
- 조회 결과 표시(데이터 존재 여부에 따라)

## 5.4 탭: 시세 진단

## 5.4.1 탭 구성

- AI 요약 영역(라벨: "시세 분석 AI 요약")
- 시세 및 여유금액 분석 영역
- 최우선 변제권 영역
- 공문서 확인하기 영역(등기부등본)

## 5.4.2 시세 및 여유금액 분석 UI 구성

## 5.4.2.1 보증금

- 고객 입력 보증금 금액 노출
- 금액 표기는 천만원 단위로 노출

## 5.4.2.2 매매 추정 시세

- 매매 추정 시세 노출
- 시세 변동 가능 안내 문구 제공
- "매매 추정 시세란?" 설명 팝업/툴팁 제공

## 5.4.2.3 기존 채무금액

- 합산 금액 노출
- 상태 판단 기준:
- 채무가 없거나 확인되지 않음: "양호"(녹색 표시)
- 채무가 있음: "확인 필요"(빨간색 표시)
- 경고 안내 제공
- 채무가 있다고 무조건 위험이 아님을 안내(규모/우선순위 확인 중요)

## 5.4.2.4 여유 금액

- 공식 노출: ① 매매 추정 시세 - ② 기존 채무금액 - ③ 보증금 = 여유금액
- 상태 판단 기준:
- 충분: "양호"(녹색 표시)
- 없음/부족: "확인 필요"(빨간색 표시)
- 여유금액이 없는 경우 경고 안내 제공
- 공동담보가 설정된 경우 추가 경고 안내 제공

## 5.4.3 최우선 변제권 UI

- 적용 가능 여부 상태 표시
- 적용 가능: "양호"(녹색 표시) 및 가능한 금액 표시
- 적용 불가: "확인 필요"(빨간색 표시) 및 "없음" 표시
- 최우선 변제권 설명 팝업/툴팁 제공

## 5.4.4 공문서 확인하기 UI

- 등기부등본 발급 기준일 노출
- 건물 등기부등본 PDF 다운로드 버튼 제공
- 토지 등기부등본 PDF 다운로드 버튼 제공

## 5.4.5 예외 표기

- 등기부등본 정보 없음: 해당 정보 없음 안내 표시
- 매매 추정 시세 정보 없음: 시세 정보 없음 안내 표시
- 공동담보 설정: 추가 경고 및 여유금액 변경 가능성 명시
- 최우선 변제권 불명확: "확인 불가" 표시

## 5.5 탭: 대출/보험 예비심사

## 5.5.1 영역 타이틀

- 타이틀: "대출과 보증보험 가입이 가능한지 주요 위험 요소를 확인해드릴게요."
- 서브 타이틀: "각종 심사 조건과 보증보험 요건을 기준으로 이 집의 구조가 어떤지 확인할 수 있도록 정리해드려요."

## 5.5.2 AI 요약 UI

- 라벨: "대출/보험 및 안전성 심사 AI 요약"
- 케이스 문구(예시):
- 긍정(가능성 높음): "이 집은 보증보험 가입과 대출 심사 모두 가능성이 높은 것으로 보여요!"
- 부정(제한 가능성 높음): "이 집은 압류·가압류 등 권리 사항으로 인해, 보증보험 가입·전세대출 모두 제한될 가능성이 높아요."
- 면책 안내 문구: "예비 심사는 실제 대출상담결과와 다를수있습니다."를 AI 요약 하단에 노출

## 5.5.3 보증보험 예비심사 UI

- 상태 배지(가능성 높음/불가) 노출
- 상태별 메시지(예시) 및 상세 설명 제공
- 가능성 높음: 녹색 배지, "보증보험 가입에 문제가 없어 보여요!"
- 불가: 빨간색 배지, "보증보험 가입이 어려울 것 같아요."

## 5.5.4 보증금 대출 예비심사 UI

- 상태 배지(가능성 높음/불가) 노출
- 상태별 메시지(예시) 및 상세 설명 제공
- 가능성 높음: 녹색 배지, "보증금 대출도 가능한 구조에요."
- 불가: 빨간색 배지, "보증금 대출이 힘들 수 있어요."

## 5.5.5 예외 표기

- 등기부등본 정보 없음: 예비심사 불가 안내 표시
- 시세 정보 없음: 정확한 판단이 어렵다는 안내 표시
- 권리관계 복잡: "확인 필요" 안내 및 전문가 상담 권장
- 보증금 정보 미입력: 예비심사 불가 안내 표시

## 5.7 탭: 치안/안전

## 5.7.1 섹션 구성

- 방범 시설 분포
- 안전 귀갓길 체크
- 동네 유흥업소 수 비교
- 지난해 범죄 발생 수 비교
- 안전한 생활을 위해 추가로 확인해보세요

## 5.7.2 방범 시설 분포 UI

- 반경 500m 기준 지도 시각화(반경 원 + 시설 위치)
- 시설 종류별 개수 리스트 노출
- 상태 태그: "양호" 또는 "확인 필요"
- 상태 표기: 상태 + 방범 시설 개수

## 5.7.3 안전 귀갓길 체크 UI

- 지도 경로 시각화 제공
- 경로 구분 표기: 위험 구간(빨간색), 주의 구간(노란색), 도로 경로(점선)
- 경로 내 방범 시설 개수 표기(방범 CCTV, 안전비상벨)
- 상태 태그: "양호" 또는 "확인 필요"

## 5.7.4 동네 유흥업소 수 비교 UI

- 비교 바 차트(인근 동네 2~3개 비교)
- 상태 태그: "양호" 또는 "확인 필요"
- Q&A 문구 제공:
- "유흥 업소에 포함된 것들에 어떤 것들이 있나요?"
- "단란주점, 나이트클럽, 룸살롱, 모텔 등을 포함합니다."

## 5.7.5 지난해 범죄 발생 수 비교 UI

- 비교 바 차트(인근 구 2~3개 비교)
- 데이터 출처 표기: "출처: 경찰청"
- Q&A 문구 제공:
- "어디서 가져온 정보인가요?"
- "1년에 한번 시/군/구 단위로 업데이트 되는 경찰청 최신 데이터 기준으로 제작되어 보여집니다."
- "비교하는 범죄는 어떤 것들이 있나요?"
- "폭력, 사기, 절도, 성폭행 등 4개의 범죄를 기준으로 비교하고 있습니다."

## 5.7.6 추가 확인 섹션 UI

- 안전한 내 집 찾기
- 제목: "안전한 내 집 찾기"
- 설명: "경찰청이 안내하는 '집 계약 전 반드시 확인해야 할 안전 체크리스트'를 바로 확인해보세요."
- 버튼: "체크리스트 보기"
- 주변 성범죄자 조회
- 제목: "주변 성범죄자 조회"
- 설명: "계약 전 집 주변 성범죄자 정보 여부를 확인할 수 있어요."
- 버튼: "조회하러 가기"

## 5.7.7 예외 표기

- 치안 관련 데이터 없음: 해당 정보 없음 안내 표시
- 지도 표시 불가: 리스트 형태로만 표시
- 비교 데이터 없음: 해당 매물/구 정보만 표시
- 지하철역 없음: "해당 없음" 표시
- 경로 계산 불가: "확인 불가" 표시

## 5.8 탭: 생활/편의

## 5.8.1 섹션 구성

- 편의 시설
- 내 동네와 비교하기

## 5.8.2 편의 시설 UI

- 반경 500m 기준 지도 시각화(반경 원 + 시설 위치)
- 시설 종류별 개수 리스트 노출
- 상태 태그: "양호" 또는 "확인 필요"
- 상태 표기: 상태 + 편의 시설 개수

## 5.8.3 내 동네와 비교하기 UI

- 내 동네 설정 버튼: "+ 내 동네 설정"
- 레이더 차트: 6개 축 비교(이 동네: 실선, 내 동네: 점선)
- 생활 인프라 설명 문구 제공(예시 문구 PRD 포함)
- 특정 항목 클릭 시 이 동네/내 동네 개수 비교 표시(예: "대중교통 - 이 동네 3개, 내 동네 183개")
- 상태 태그: "양호" 또는 "확인 필요"

## 5.8.4 예외 표기

- 편의 시설 데이터 없음: "확인 불가" 표시 또는 해당 정보 없음 안내 표시
- 지도 표시 불가: 리스트 형태로만 표시
- 내 동네 미설정: 내 동네 설정 버튼만 표시, 비교 차트 비노출
- 내 동네 데이터 없음: 해당 항목 "확인 불가" 표시
- 비교 데이터 없음: 해당 매물 정보만 표시
